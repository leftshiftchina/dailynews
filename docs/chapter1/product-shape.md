# 第一章 产品形态

## 产品定位

每日新技术推送小博客定位为轻量、专注、移动优先的每日技术小报。它不追求复杂的信息流，而是每天稳定交付一篇可读、可回看、可沉淀的技术内容。

## 核心页面

| 页面 | 功能 |
| --- | --- |
| 今日页 | 展示当天标题、摘要、标签、正文、来源和发布时间 |
| 历史页 | 按日期倒序展示历史新闻列表 |
| 详情页 | 查看某一天的完整内容 |
| 设置页 | 展示 PWA 安装引导、通知订阅入口和版本信息 |

## 今日页

今日页是默认入口，建议采用单列阅读布局：

1. 日期和阅读时间。
2. 标题。
3. 一句话摘要。
4. 标签。
5. Markdown 正文。
6. 来源和更新时间。

移动端样式建议：

| 项 | 建议 |
| --- | --- |
| 最大内容宽度 | `720px` |
| 左右边距 | `16px` |
| 正文字号 | `16px` 到 `17px` |
| 正文行高 | `1.7` 到 `1.8` |
| 标题字号 | `24px` 到 `30px` |

## 历史页

历史页主要解决回看问题：

- 按日期倒序展示。
- 每条展示标题、摘要、标签和阅读时间。
- 点击后进入详情页。
- 第一版使用分页即可，后续可以升级为无限滚动。

## 设置页

设置页不需要做得很重，第一版建议包含：

- 当前版本。
- PWA 安装状态。
- 最近更新时间。
- 通知订阅能力检测。

通知提醒建议放到第二阶段，因为不同浏览器和系统对 Web Push 的支持差异比较明显，特别是 iOS 需要用户把 Web App 添加到主屏幕后才能获得完整体验。
