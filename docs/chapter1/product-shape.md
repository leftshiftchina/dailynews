# 内容形态

## 站点定位

AI 前沿科技简报定位为轻量、专注、可长期维护的内容分享站。它不追求复杂的信息流，而是把有价值的 AI 科技资讯、工具观察、实现教程和部署经验沉淀成可以反复阅读的文档。

## 核心内容

| 内容 | 说明 |
| --- | --- |
| 首页 | 说明项目定位、阅读入口和主要内容 |
| 项目导读 | 介绍架构、目录、快速开始和搭建方式 |
| AI简报 | 按日期归档每日 AI 前沿科技内容 |
| 内容建设 | 说明栏目规划、写作结构和站点组织方式 |
| 部署手册 | 记录 GitHub Pages 发布流程和常见问题 |

## AI简报页面

AI简报是这个站点最重要的持续内容。建议每篇简报保持稳定结构：

1. 日期。
2. 标题。
3. 今日要点。
4. 分主题内容。
5. 原始来源链接。
6. 简短点评或后续观察。

推荐文件命名：

```text
docs/daily/2026-06-23.md
```

日期统一使用 `YYYY-MM-DD`，这样在侧边栏、搜索和后续归档中都更清晰。

## 专题页面

当某个方向值得持续跟踪时，可以把多篇 AI简报整理成专题，例如：

- AI Agent 工具链。
- 多模态模型进展。
- 开源模型与推理框架。
- AI 编程与软件工程。
- 企业 AI 应用案例。

专题页适合放在新的目录中，例如：

```text
docs/topics/agent-tools.md
docs/topics/open-models.md
```

## 教程页面

教程页面适合记录可复用的实现过程。建议采用：

- 背景：为什么要做。
- 准备：需要什么环境。
- 步骤：一步一步完成。
- 验证：如何确认成功。
- 常见问题：容易踩坑的位置。

这个仓库中的 [自己搭建教程](/guide/build-your-own) 就是一个基础示例。
